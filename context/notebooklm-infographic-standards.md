# Header

Read before creating NotebookLM source material for Natively infographics, slide decks, or other visual artifacts.

# Content

NotebookLM visual sources should be written for narrative impact. The source should feel like a polished marketing announcement, executive update, or press-release-style brief. It should translate technical work into business value, product promise, proof points, and future-state momentum.

Do not include file paths, repo paths, command names, package names, implementation filenames, or dry code-changelog inventory unless Steve explicitly asks for a technical source. Keep enough factual detail for NotebookLM to stay grounded, but write the material so it is enjoyable to read on its own.

Before generating an infographic, use NotebookLM's Reports output as a prompt discovery step when possible. Capture all returned suggested report formats, usually four, including each title, description, and prompt. Review the full set of source-derived angles, inspect the generated report prompt for the best suggestion, and use that NotebookLM-authored prompt as the infographic custom prompt when it is useful. If no useful NotebookLM-generated report prompt is available, skip the custom prompt rather than inventing one.

# Learnings

- [2026-05-12] NotebookLM infographic sources should read like polished marketing or press-release material, not technical implementation documents.
- [2026-05-12] NotebookLM infographic generation should usually skip the custom prompt when the source and visual references are strong; never prescribe the exact theme, creative storyline, narrative structure, layout, or visual sequence.
- [2026-05-12] Steve prefers using NotebookLM's dynamic Reports suggestions as prompt discovery: inspect the suggested report prompt, then generate the infographic using that NotebookLM-authored prompt when it is useful.
- [2026-05-12] Default NotebookLM infographic workflow should capture all returned suggested report prompts, usually four, so the source-derived narrative angles are all visible before selecting the infographic prompt.
- [2026-05-28] Fully processed meetings over 10 minutes should generate portrait, detailed NotebookLM infographics as a non-blocking artifact job; manual or unexpected sessions should not be skipped solely because they are not calendar-backed.
