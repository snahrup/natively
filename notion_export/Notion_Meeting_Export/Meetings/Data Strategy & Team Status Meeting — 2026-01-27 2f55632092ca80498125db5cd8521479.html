Summary

Microsoft Fabric Implementation Overview

  • Steve outlined the comprehensive Microsoft Fabric implementation project across the entire organization ‣
    • Goal is to centralize all data from various sources into a single location accessible to all divisions and departments ‣
    • Will enable real-time or near real-time data access without relying on standard SSRS reports
    • Executives like Mike Spencer need a 360-degree view of data across all departments, which is currently not feasible ‣
    • Fabric will serve data scientists, data analysts, data engineers, business analysts, and regular business users in a single interface ‣
    • Plans to layer in Copilot as an AI interface for data interaction and report creation ‣

Microsoft Purview Documentation Layer

  • Documentation and data governance will be handled through Microsoft Purview (being folded into Fabric) ‣
  • Will provide comprehensive data lineage tracking from source to reports ‣
    • Track how data transforms through ETLs
    • View formula changes and definition modifications throughout the data journey
    • Enable impact analysis before making changes to understand downstream effects ‣
  • Will identify data stewards and owners for proactive communication when changes are needed ‣
  • Purview implementation will run in parallel with Fabric deployment ‣

Master Data Management (MDM) Strategy

  • MDM initiative aims to establish common terminology and processes across the organization ‣
  • Key focus on defining system of record and standardizing calculations ‣
    • Currently, different entities calculate metrics differently
    • Need to establish where formulas live and what the official system of record is ‣

Citrine Project and R&D Integration

  • Ann clarified that Citrine is a tool for R&D chemists, not a database or system of record ‣
    • Used for formulation work and machine learning/modeling
    • Data temporarily stored in Citrine during development but not intended as a repository ‣
  • Current challenge: source data for Citrine is scattered across multiple locations with unclear access ‣
  • Citrine and Fabric initiatives are complementary - Fabric can provide the organized data sources that Citrine needs ‣
  • Costing workbench discussion: need to address how to keep costing data current in Citrine, as costs constantly change ‣ ‣
