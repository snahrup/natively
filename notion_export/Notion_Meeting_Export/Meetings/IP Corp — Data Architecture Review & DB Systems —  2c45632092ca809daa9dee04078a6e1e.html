Database Systems Overview

  • The organization maintains multiple database servers with specific purposes [14]:
    • Main production servers: M3DB1, M3DB2, M3DB3, SQL 2016 Live, and SQL 2019 Live [15] [16]
    • Development and test environments make up most of the other servers [15]
    • Some servers are scheduled for retirement [16]
  • M3DB3 hosts the on-premise ERP system (M3) [17]
  • M3DB1 hosts both M3 and MES applications [18]
  • Data is log-shipped to SQL 2012 test server which serves as a staging environment for report writing [19] [20]

Data Architecture Discussion

  • Current state: Transactional databases without true data warehouse functionality [21]
  • The team has created some "mini data warehouses" for reporting purposes [22] [23]
  • Current ETL process involves pulling data from source systems into SQL reporting databases [24] [25]
  • Data refreshes are currently implemented using full truncation and reload with date-based filtering [26] [27] [28]
  • Discussion around implementing a medallion architecture (bronze/silver/gold layers) in Microsoft Fabric [29] [30] [31]
  • Udias is currently using star schema for simplicity in Power BI reporting [32] [33]

    • ION: Infor middleware routing BODs between M3 and other apps, used mainly for troubleshooting
    • ETQ: Quality, HSE, and incident management app, feeds data into M3 but not primary source

    • SDS, PCD, ANV: safety data sheets, process‑controlled data, and MES extension tables

    Data Flow & Integration

    • Raw‑material call‑outs drive consumption entries back into M3 (inventory reduction)
    • PLC data posted directly to M3 for accurate inventory; manual entries allowed with variance codes
    • MES passes batch‑level data to M3 for inventory updates only (no costing updates)
    • ION can expose logs/monitoring data for troubleshooting; frequency of use to be clarified
    • ETQ receives quality/HSE incidents from M3 and other systems, not a primary data source
    • Advanced Scheduler pulls resource and batch data from M3/MES to build production schedules

    Acronyms & Terminology

    • ANV – MES extension table (viscosity/acid number tracking)
    • PCD – Process Controlled Data (PLC‑related charge info)
    • SDS – Safety Data Sheets (material safety info)
    • RPT – Report (generic)
    • NAC – North American Composites (distributor role)
    • MPD/SHAR – Molding Products division, connects to M3 Cloud, no MES layer
    • OAT/OATS – Operational Acceptance Testing

    Medallion Architecture & Fabric Strategy

    • Bronze layer: raw source tables landed unchanged (e.g., M3 DB extracts)
    • Silver layer: transformed joins, initial business logic, serves as reporting sandbox
    • Gold layer: fully modelled star schema for production Power BI reports, minimal custom queries
    • Incremental refresh logic needed; confirm CDC/change‑tracking on source DBs
    • Consider Dataflows Gen2 + Lakehouse tables to replace nightly SQL reloads

    Database Landscape & Access

    • Production DBs: M3 DB 1‑3 (on‑prem ERP), SQL 2016 Live, SQL 2019 Live (core apps)
    • Staging/Read‑only copy: SQL 2012 Test (used for reporting)
    • Development/Test DBs: M3 Dev DB 1‑3 (sandbox)
    • Current "mini‑warehouse" in SQL acts as a de‑facto silver layer for Power BI
