Summary

Project Scope and Rebranding

  • The team discussed rebranding the initiative from "Proof of Concept" to "MVP" (Minimum Viable Product), recognizing that the decision to move forward with Microsoft Fabric has already been made ‣ ‣
  • Microsoft Fabric will serve as a comprehensive solution encompassing data warehousing, ETL pipelines, data transformation, cleansing, and reporting capabilities ‣ ‣ ‣
  • The solution is designed to consolidate data from disparate databases into a single lakehouse/warehouse location ‣

Microsoft Fabric Capabilities

  • Fabric handles the full data lifecycle including data curation, preparation, and master data management ‣
  • The platform supports creating measures, establishing naming conventions, and preparing data for reporting through various methods including Fabric notebooks and data flows ‣
  • Microsoft Purview will be implemented in parallel to provide documentation, cataloging, and data dictionaries ‣ ‣
  • Purview offers a searchable interface for finding data, definitions, and metadata at a granular level ‣ ‣

Data Governance and Stewardship

  • The initiative includes identifying and documenting data stewards across the organization who will have autonomy over data decisions within their domains ‣ ‣
  • Data stewards will serve as points of contact for questions about specific data types and will be responsible for key decisions ‣
  • Documentation must be fluid and maintained through a centralized hub ‣

Data Architecture Approach

  • The team will implement a bronze/silver/gold layer architecture ‣ ‣
  • Bronze layer: Raw data dumped as-is from source systems without alteration, providing a perfect reflection of source data ‣
  • Silver layer: Transformed data with contextual joins (e.g., combining MES with M3 ERP data) ‣ ‣
  • Gold layer: Final reporting-ready data (structure still being determined) ‣

Data Domain Strategy

  • Mike emphasized designing around data domains rather than business structure, as organizational structure may change ‣ ‣
  • The distinction between manufacturing and distribution is expected to remain stable long-term ‣ ‣
  • Product data standardization across multiple systems (pre-cloud, in-plastic, NAC) was identified as a challenge ‣ ‣

Starting Point Decision

  • Steve proposed starting with MES (Manufacturing Execution System) data as it serves as the foundation for other systems ‣
  • The initial implementation will focus on MES and M3 ERP integration ‣ ‣
  • Mike noted that MES will undergo future replacement, but agreed to proceed with it as the starting point ‣ ‣
  • The team acknowledged that starting with any foundational source will inevitably touch multiple areas ‣

MVP Objectives

  • Primary goal is to demonstrate the ability to standardize and classify data into domains, bring it into a central location, and show value ‣
  • The initiative aims to establish data cleansing practices and governance to prevent "garbage in, garbage out" scenarios ‣
  • Quick wins are expected from OT data correlation and integration ‣
  • Bronze and silver layers will clearly reveal data quality issues and inconsistencies across source systems ‣ ‣

Implementation Philosophy

  • The team agreed to start implementation understanding they may only know 70% of requirements, with the remaining 30% becoming clear during execution ‣
  • Launching the system will reveal necessary adjustments and refinements ‣
  • Domain definition may be more appropriate at the reporting layer rather than in the bronze/silver data layers ‣

Key Considerations

  • Master source systems need to be defined for each data type ‣ ‣
  • Product classes and naming standardization will need to be addressed in source systems, not just in the data fabric ‣
  • Future MES replacement and inventory management changes (moving to Factory Track) will impact the data structure ‣ ‣
  • Microsoft Purview must be implemented alongside Fabric to avoid retroactive documentation challenges ‣

Action Items

  • Steve to send follow-up agenda ‣
  • Team to officially rebrand from POC to MVP terminology ‣
  • Define and document data domains ‣ ‣
  • Identify and assign data stewards for each domain ‣ ‣
  • Stand up Microsoft Purview in parallel with Fabric implementation ‣
  • Begin with MES and M3 data integration as the starting point ‣ ‣
  • Create visualization of the bronze/silver/gold layer approach ‣
